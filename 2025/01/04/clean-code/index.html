Clean Code Best Practice

Created2025-01-04|Updated2025-01-04|Code Design
|Post Views:

Introduction

This article is to introduce some code style best practices.

  • Target：Write clean and maintainable code.

Function

Function 1：Parameters

Having >= 3 parameters in a function:

  • Increases code complexity and reduces readability.
  • Makes unit testing more difficult to maintain.

Thus, >= 3 parameters should be treated as a wrong case. Instead, we do the followings:

  1. Combine related parameters into a class
    • Group related parameters to reduce the number of function arguments.
  2. Refactor the function logic
    • Split the function into smaller, more manageable functions.

Function 2: Single Responsibility Principle

Each function should only care about and do one thing.

  1. Split the function into smaller, focused functions
    • Break the function into individual tasks, each handled by its own function.
  2. Use clear and descriptive names
    • Ensure the function name reflects its single purpose.
